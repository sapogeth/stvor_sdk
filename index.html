Why STVOR

Production-grade encryption.
Developer-grade API.

Everything Signal uses — packaged for developers who want results, not a PhD.

🔐

Signal Protocol

Full X3DH key agreement + Double Ratchet with one-time prekeys. The same cryptography as Signal, WhatsApp, and Wire — forward secrecy from the first message.

⚛️

Post-Quantum ready

ML-KEM-768 (NIST FIPS 203) implemented from scratch — no external libraries. Verified against official NIST ACVTS test vectors. Enable with pqc: true.

⚡

Zero dependencies

Node.js SDK uses only built-in node:crypto. Browser uses Web Crypto API. No WASM, no npm bloat, no supply chain risk.

👥

Group chats

Sender Keys scheme — O(1) encryption regardless of group size. Auto-ratchets sender key on member removal. Same API as 1-to-1.

📦

Any JS type

Send strings, numbers, objects, Dates, Sets, Maps, Uint8Arrays — original type is preserved end-to-end. No manual serialization.

🌐

Browser + Node.js

Full Double Ratchet in the browser via Web Crypto API. Same API, same security. Keys persist in IndexedDB across page refreshes.

How it works

Simple API, serious crypto.

Three steps. No cryptography expertise required.

1

Install & start mock relay

npm install @stvor/sdk then npx @stvor/sdk mock-relay — local relay on port 4444, no account needed.

2

Connect a user

Stvor.connect({ userId, appToken, relayUrl }) — generates identity keys, registers with relay, starts polling. One call, fully automatic.

3

Send & receive

alice.send('bob', data) — X3DH + Double Ratchet happens automatically. Recipient gets the message via onMessage, type preserved.

# Stvor SDK
End-to-end encryption for any app. Signal Protocol (X3DH + Double Ratchet). Zero runtime dependencies.
Install
npm install @stvor/sdk
Quickstart
import { Stvor } from '@stvor/sdk'; const alice = await Stvor.connect({ userId: 'alice', appToken: 'stvor_live_xxx', relayUrl: 'https://relay.stvor.xyz', pqc: true, }); bob.onMessage(msg => console.log(msg.from, msg.data)); await alice.send('bob', { text: 'Hello!' }); await alice.disconnect();
Stvor.connect(config)
userIdstring — any unique identifier
appTokenstring — must start with 'stvor_'
relayUrlstring — 'https://relay.stvor.xyz' or self-hosted
pqc?boolean — ML-KEM-768 hybrid, default: false
sealedSender?boolean — hide sender from relay, default: false
timeout?number — ms, default: 10000
