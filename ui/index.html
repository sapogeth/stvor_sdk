STVOR SDK

Honest Signal-style E2EE SDK for startups and pilots

STVOR is a client-side end-to-end encryption SDK inspired by the Signal Protocol. It implements X3DH key exchange and Double Ratchet for forward secrecy.

For: Startups, pilot projects, internal tools that need honest, transparent E2EE.

Not for: Enterprise environments, high-risk applications, or scenarios requiring post-quantum security.

Installation

Install the SDK in your client application. The server never participates in encryption.

The relay only forwards ciphertext and cannot decrypt messages.

npm install @stvor/sdk

Quick Start

Copy this code to start using STVOR. The token is automatically inserted when you generate one above.

import { Stvor } from '@stvor/sdk';

const app = await Stvor.init({
  appToken: 'YOUR_TOKEN_HERE'
});

const alice = await app.connect('alice@example.com');
await alice.send('bob@example.com', 'Hello Bob!');

Security Model

‚úÖ What STVOR guarantees

  • End-to-end encryption (plaintext never leaves the device)
  • Forward Secrecy via Double Ratchet
  • Post-Compromise Security after an honest DH ratchet
  • Replay protection
  • Authenticated encryption (AEAD)

‚ùå What STVOR does NOT guarantee

  • MITM protection on first contact (TOFU model)
  • Metadata privacy ‚Äî relay sees sender, recipient, timing
  • Guaranteed PCS if DH messages are blocked
  • Post-quantum security
  • Multi-device support

Threat model summary

  • Relay is honest-but-curious
  • Clients must store session state securely
  • PCS requires at least one honest DH exchange

Limitations & Positioning

  • STVOR is not an enterprise product
  • Designed for startups, pilots, and internal tools
  • No GDPR, SOC 2, or compliance guarantees
  • All limitations are documented honestly
STVOR is a reference implementation. For production use, ensure you understand the threat model and limitations.
